\section{The C++/Rcpp reimplementation of the RTIGER and nNIL cores}
\label{supp:optimization}
%======================================================================

\begin{sloppypar}\hbadness=10000\relax
The named callers of the main text run on a single engine reimplemented in
C++/Rcpp (\code{nilhmm/src/}: \code{rtiger.cpp}, \code{viterbi\_batch.cpp},
\code{emission\_count.cpp}, \code{lbimpute.cpp}, \code{fsfhap.cpp}, and the shared
forward/backward and segment routines). This reimplementation removes the Julia
runtime that upstream RTIGER \citep{rtiger_2023} requires through
\code{JuliaCall}, and it makes a single fit cheap enough to loop over the
calibration sweeps and coverage sweeps of the main text at $10^4$ to $10^5$
markers on a laptop. We first establish its correctness, then report speed and
memory.
\end{sloppypar}

We benchmark the shipped C++/Rcpp core directly against the \emph{upstream
original} Julia RTIGER \citep{rtiger_2023} on one shared marker panel: a
Viterbi-identical equivalence oracle first, then throughput and memory. The
original Julia is the baseline (its 109{,}703-marker convergence fit takes about
3.4~hours).

\paragraph{Data.} Real \emph{Arabidopsis} Col-0 $\times$ Ler allele counts
(BioProject PRJNA720439 / SRA study SRP313911), the 109{,}703-locus panel of loci
covered in all three of the BN/Z/AU samples, thinned by odd index to five sizes
(6{,}857; 13{,}713; 27{,}426; 54{,}852; 109{,}703 markers per sample). Both cores
were run from the identical deterministic initialization ($\alpha=[20,20,1]$,
$\beta=[1,20,20]$, diagonal-dominant transition, uniform start), rigidity~2,
\code{eps}~$=0.01$. Compute: \code{scripts/bench\_rtiger\_cpp\_vs\_julia.R}. The
nNIL reproduction (Sec.~\ref{supp:nnil}) uses the Zhong \emph{et al.} nested-NIL
panel.

\paragraph{Environment.} All timings are wall-clock on an Apple M4 (10 cores,
24~GB RAM), macOS~26.5 (\code{arm64}), single-threaded (\code{threads}~$=1$). The
nilHMM C++/Rcpp core was built native \code{arm64} with Apple clang~17
(\code{-std=gnu++17 -O2}); R~4.5.2, nilHMM~0.3.0, Rcpp~1.1, RcppParallel~5.1.11.
Its forward/backward, Viterbi, and emission M-step are hand-written scalar C++
(\code{LinkingTo}: only Rcpp and RcppParallel; no Armadillo, Eigen, or external
BLAS/LAPACK), so the system linear-algebra library cannot affect these timings.
Throughput points are the median of three fits. The upstream-original Julia
RTIGER baseline (Julia~1.12.6) was run on the same hardware; its 109{,}703-marker
convergence figures are read from those preserved runs. The full environment is
recorded per run in \code{results/bench/rtiger\_env.txt}.

%----------------------------------------------------------------------
\subsection{Equivalence to Julia RTIGER (correctness first)}
%----------------------------------------------------------------------
Both cores were run to full convergence from identical deterministic
initialization on the shared panel at all five sizes, and their outputs compared
position by position (Table~\ref{tab:cpp-equiv}). At every size the two cores take
the \emph{same} number of EM iterations, impute \emph{identical} ancestry states
at every marker position across all three samples (0 mismatches; 329{,}109 of
329{,}109 states at the full panel), and fit emission parameters agreeing to
$\le 8\times10^{-6}$, which is float-summation order, not an algorithmic
difference. The per-iteration convergence trajectories coincide on the
$1{:}1$ line (Fig.~\ref{fig:cpp-equiv}, right).

%----------------------------------------------------------------------
\subsection{Throughput}
%----------------------------------------------------------------------
On a log-log plot the per-iteration wall time against markers per sample is a
straight line whose slope is the empirical complexity exponent
(Fig.~\ref{fig:cpp-scaling}). The upstream original scales as
markers$^{2.10}$, roughly quadratic, dominated by an emission M-step that rebuilds
a \code{BetaBinomial} per marker on every optimizer evaluation. The C++/Rcpp core
scales as markers$^{0.96}$, essentially linear, because it groups markers by
distinct read-count pair $(k,n)$ and pre-sums the $\gamma$ weights once, so the
emission line search costs $O(\#\text{distinct pairs})$ rather than
$O(\#\text{markers})$. Removing that one factor of $N$ is what turns the full fit
from hours into a second and makes the calibration and coverage sweeps of the main
text tractable (Table~\ref{tab:cpp-speed}).

(linear).}

%----------------------------------------------------------------------
\subsection{Memory}
%----------------------------------------------------------------------
The C++ E-step folds each chain into fixed-size sufficient statistics (the
$s\times s$ transition counts, the start and emission accumulators, and a map over
the distinct $(k,n)$ pairs) and reduces them in a fixed chain order; it never
retains the per-sample \code{zeta}/\code{gamma}/\code{alpha}/\code{beta} arrays.
EM scratch is therefore $O(\text{threads}\cdot T_{\max})$ rather than
$O(N\cdot T)$: it does not grow with the sample count $N$. A peak-RSS sweep
confirms this (Table~\ref{tab:cpp-mem}): cycling 2 to 60 samples at 50{,}000
markers each, peak resident memory grows as only
$\sim$162~MiB~$+\;2.1$~MiB/sample ($R^2=0.97$), where the intercept is the
R and package runtime and the $2.1$~MiB/sample slope is the retained input count
vectors, not EM working memory. There is no per-sample array retention, so a
population-scale joint fit stays within a workstation's memory.

\section{nNIL reproduction and the calibration foil}
\label{supp:nnil}
%======================================================================
nilHMM's \code{nNIL} caller (categorical genotype-confusion emission, geometric
duration) reimplements Holland's original nested-NIL HMM \citep{zhong2025}. Two
results: (i) run on the Zhong \emph{et al.} panel it reproduces the published
introgression calls \todo{marker concordance / donor-segment Jaccard}; (ii) on the
24-line SNP-chip-truth subset, the operating point Zhong \emph{et al.} selected by
chip optimization and the \code{rrate} selected by \code{simcross} calibration (no
chip) coincide \todo{show the two operating points and their segmentation
agreement}. Result (ii) is the concrete population-scale evidence for the main
text's claim that simulation substitutes for scarce high-coverage truth. Compute:
\code{analysis/nnil-reproduction-benchmark.qmd}.
